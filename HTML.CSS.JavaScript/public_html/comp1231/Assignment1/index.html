Advanced DOM Manipulation

What is Document Object Model(DOM)

The Document Object Model is a platform- and language-neutral interface that will allow programs and scripts to dynamically access and update the content, structure and style of documents. The document can be further processed and the results of that processing can be incorporated back into the presented page.

Getting an element

To manipulate DOM using JavaScript, the appropriate part of the tree must be selected.

It can be an individual element. All is needed is to tell JavaScript what it is, which is usually done by using document.getElementById.

document.getElementsByTagName works in much the same way as getElementById, except that it takes a tag name (a,ul,li, etc) instead of an ID and returns a NodeList, which is essentially an array of the DOM Elements.

document.getElementsByClassName returns the same kind of NodeList as getElementsByTagName, except that you pass a class name to be matched, not a tag name.

createTextNode() and createElement()

The createTextNode() method creates a Text Node with the specified text.

By calling createElement() we can create a new element with the argument being the element's HTML tag name.

appendChild()

The appendChild() method appends a node as the last child of a node.

Using this method and the methods above, we can create new elements and put them inside the document

removeChild()

The removeChild() removes the specified child node from current element.

The specified child node must be an immediate child of the current element

Removed child is no longer present in the document tree, but with the reference returned by the removeChild method it can be inserted later into any element in the same document.
